Como funciona a autenticação

A API interna do Chess.com (usada para torneios e estatísticas avançadas) requer cookies de sessão — não OAuth. Por isso, as credenciais são configuradas uma vez no servidor via arquivo .env.

Todos os visitantes do portal compartilham essas credenciais — não é necessário que cada usuário faça login individualmente.

Passo a passo
  1. Instale a extensão chessclub Cookie Helper (Chrome/Edge) a partir do código-fonte da biblioteca:
    chessclub/tools/chessclub-cookie-helper/
    Abra chrome://extensions, ative o "Modo do desenvolvedor" e clique em "Carregar sem compactação".
  2. Faça login no Chess.com no mesmo navegador.
  3. Clique no ícone da extensão na barra de ferramentas e copie os valores de ACCESS_TOKEN e PHPSESSID.
    Alternativamente: abra as DevTools do Chrome (F12) → Application → Cookies → chess.com e copie manualmente.
  4. Cole os valores no arquivo .env do servidor: 
    CHESSCOM_SERVER_ACCESS_TOKEN=seu_access_token_aqui
CHESSCOM_SERVER_PHPSESSID=seu_phpsessid_aqui
  5. Reinicie o servidor (python run.py) para carregar as novas credenciais.
Observações importantes
  • ACCESS_TOKEN expira em ~24 horas — será necessário repetir o processo diariamente para dados autenticados.
  • PHPSESSID tem validade maior (~2 semanas).
  • Use a conta de um membro do clube para obter os melhores resultados.
  • Os cookies ficam somente no arquivo .env do servidor — nunca são expostos ao navegador do visitante.
