How authentication works

The Chess.com internal API (used for tournaments and advanced statistics) requires session cookies — not OAuth. Therefore, credentials are configured once on the server via the .env file.

All portal visitors share these credentials — individual user login is not required.

Step by step
  1. Install the chessclub Cookie Helper extension (Chrome/Edge) from the library source code:
    chessclub/tools/chessclub-cookie-helper/
    Open chrome://extensions, enable "Developer mode", and click "Load unpacked".
  2. Log in to Chess.com in the same browser.
  3. Click the extension icon in the toolbar and copy the ACCESS_TOKEN and PHPSESSID values.
    Alternatively: open Chrome DevTools (F12) → Application → Cookies → chess.com and copy manually.
  4. Paste the values into the server's .env file: 
    CHESSCOM_SERVER_ACCESS_TOKEN=your_access_token_here
CHESSCOM_SERVER_PHPSESSID=your_phpsessid_here
  5. Restart the server (python run.py) to load the new credentials.
Important notes
  • ACCESS_TOKEN expires in ~24 hours — you will need to repeat the process daily for authenticated data.
  • PHPSESSID has a longer validity (~2 weeks).
  • Use a club member's account for best results.
  • Cookies are stored only in the server's .env file — they are never exposed to the visitor's browser.
